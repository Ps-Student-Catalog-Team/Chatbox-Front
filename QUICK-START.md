## 🎮 功能演示

### 登录后
1. **自动加载聊天记录** ✨

   - 显示所有群成员
   - 标注群主身份 👑

#### 作为普通成员
点击**右上角 ⋮ 菜单** → 选择操作：

1. **👥 查看成员列表**
   - 查看群中所有成员

2. **🚪 退出群聊**
   - 从群聊中移除自己

点击**右上角 ⋮ 菜单** → 选择操作：

1. **🗑️ 清空聊天记录**

## 🆘 常见问题

### Q: 私聊记录为什么不显示？
**A**: 
1. 确保已登录
2. 如果仍未显示，刷新页面
3. 确认后端服务运行正常

### Q: 邀请成员为什么显示"所有好友都已是成员"？
**A**: 
- 这是正常的，说明所有好友都已加入该群聊
- 如需添加新成员，先通过"通讯录"→"添加好友"添加新好友

### Q: 修改群名为什么其他成员看不到？
**A**: 
1. 检查网络连接是否正常
2. 其他成员可能需要刷新来获取最新群名

### Q: 怎样确认我是群主？
**A**: 
- 如果能看到 **⚙️ 群聊设置** 选项，说明你是群主
- 在成员列表中，群主会显示 👑 标记

### Q: 解散群聊后能恢复吗？
**A**: 
- 不能恢复，所有群聊和消息都会被删除
- 请确保确实需要解散才点击确认

## 🔍 调试技巧

### 查看控制台日志
按 **F12** 打开开发者工具，选择 **Console** 标签：

**正常日志应该包含**:
```
[聊天记录] 已自动加载 X 个私聊和 Y 个群聊的历史消息
```

**错误日志会显示具体问题**:
```
Warning: 加载历史消息失败
Error: 获取群成员失败
```

### 查看网络请求
在 **Network** 标签中查看：
- `/api/messages` - 加载聊天记录
- `/api/group/{id}/members` - 获取群成员
- `/api/online-users` - 获取在线人数
- `/ws` - WebSocket连接

### 后端日志
服务器启动时会输出：
```
局域网聊天室已就绪，启动于 :40001 ...
```

## 🎓 代码实现

### WebSocket消息格式
所有新增操作都通过WebSocket传输：

**邀请成员**:
```json
{
  "action": "add_member_to_group",
  "group_id": "123",
  "members": ["user1", "user2"]
}
```

**修改群名**:
```json
{
  "action": "rename_group",
  "group_id": "123",
  "new_name": "新群名"
}
```

**发布公告**:
```json
{
  "action": "publish_announcement",
  "group_id": "123",
  "announcement": "公告内容"
}
```

**解散群聊**:
```json
{
  "action": "disband_group",
  "group_id": "123"
}
```

**退出群聊**:
```json
{
  "action": "quit_group",
  "group_id": "123"
}
```
